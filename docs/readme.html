Usage Notes

  1. Because this plugin makes modifications to the Zen Cart order process, it is not directly compatible with other plugins that also make order-related modifications including (but not limited to):
    • Checkout Without Account (COWOA)
    • Edit Orders
    • Fast and Easy Checkout
    • One-Page Checkout
    • Super Orders
    • Ty Package Tracker

    If you have a question about another plugin's compatibility, please ask on this plugin's support thread (see above).

Overview

This plugin enables your store to offer your customers multiple shipping addresses on a single order and has been developed and tested using Zen Cart 1.5.5f and Zen Cart 1.5.6b using a clone of the responsive_classic template.

Every effort has been made to reduce the core-file changes required. Where possible, the change has been limited to the insertion of one or more Zen Cart notifiers at significant points. When it was not possible, the changes are spanned by comments indicating the beginning (//-bof-multiship) and end (//-eof-multiship) of the code-fragments.

  • Template-override changes are based on the responsive_classic for Zen Cart 1.5.6b.

Zen Cart Admin Interfaces

The first time an admin signs into (or refreshes the screen of) your Zen Cart admin after you have installed the files for Multiple Ship-to Addresses, an initialization script runs that creates new tables in your Zen Cart database:

  1. orders_multiship. This table holds the information that identifies the delivery address for a multiship sub-order.
  2. orders_multiship_totals. This table holds the order total's values associated with a multiship sub-order.

and modifies your orders_products table to add a field to provide the correlation to the multiship order id for each product. Entries are created in the admin_pages table to support the use of the invoice_multiship and packingslip_multiship pages as well as the plugin's addition to the configuration menu.

Additional documentation can be found online in the plugin's GitHub repository.

Zen Cart Customer Interfaces

This plugin modifies the interfaces presented to your customers during the checkout process and, if an order is shipped to multiple addresses, their account history display for that order. See the Multiple Shipping Addresses' GitHub repository for additional documentation.

Account — Reviewing Order History

When a customer has placed an order with Multiple Ship-To Addresses, that order's display within the customer's account-related pages changes to reflect the multiple addresses:

  1. On the main My Account page, the Ship To column of the Previous Orders table shows "Multiple Addresses".
  2. On the My Account :: History page, the Shipped To field shows "Multiple Addresses".
  3. On the Order Information page, the Delivery Address shows "Multiple Addresses" and the order's details are broken out by ship-to address.

Install and Un-install Instructions

Installation

There are core-file overwrites and template-overrides required by this plugin; you should always backup your cart's database and files prior to making any changes. Unzip the plugin's package file and then:

  1. Rename the /YOUR_ADMIN sub-directories to match the name of your Zen Cart's admin sub-directory.
    1. /zc155/YOUR_ADMIN
    2. /zc156/YOUR_ADMIN
  2. Rename the YOUR_TEMPLATE sub-directory to match the name of your Zen Cart's custom template:
    1. /includes/templates/YOUR_TEMPLATE
  3. Choose the core-file overwrites to be merged with your store's current file-set, based on the Zen Cart version to be used:
    • Zen Cart 1.5.6 through 1.5.6b
      1. /zc156/YOUR_ADMIN/orders.php. This is the zc156b version of the file, with 4 marked change-sections.
    • Zen Cart 1.5.5 through 1.5.5f
      1. /zc155/includes/modules/order_total/ot_shipping.php. This is the zc156 version of the file, with an additional notifier required by the plugin.
      2. /zc155/YOUR_ADMIN/orders.php. This is the zc155f version of the file, with 8 marked change-sections.
  4. For each of the template-override files distributed by this plugin, merge the plugin's changes with any other changes you've made and then copy that file to your store's file-system. In all cases, Zen Cart 1.5.6b's responsive_classic template serves as the change-basis for these files.

Uninstall

First, delete the files that were copied to your Zen Cart installation's file-system (restoring the prior contents of the core and template-override files) then use your admin's Tools->Run SQL Patches to run the file uninstall_multiship.sql that ships with this plugin.
